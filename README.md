# Nexspro

Nexspro is a proprietary digital platform built with Ruby on Rails and Phlex.

The project is designed as a long-term product foundation: Ruby-first, server-rendered, visually precise, maintainable, and intentionally independent from JavaScript frontend frameworks.

This repository is publicly visible for technical review and project presentation only.

It is not open-source.  
It is not a starter kit.  
It is not a template.  
It is not available for reuse.

## Overview

Nexspro is being developed as a large-scale digital product with a strong technical foundation and a distinct visual identity.

The project focuses on clarity, durability, performance, and a cohesive user experience. Its architecture is shaped around Rails conventions, Ruby-native views, and a disciplined interface system.

The goal is not to showcase isolated pages or reusable patterns, but to build a complete product whose code, design, structure, and identity evolve together.

## Technical Foundation

Nexspro uses Ruby on Rails as the application framework and Phlex as the primary view layer.

The interface is written in Ruby through Phlex components and views, keeping rendering logic close to the application while avoiding the complexity of a separate frontend application.

The project currently relies on:

- Ruby on Rails
- Phlex / Phlex Rails
- PostgreSQL
- Tailwind CSS
- Propshaft
- Solid Queue
- Solid Cache
- Solid Cable
- RSpec
- RuboCop
- Brakeman
- Bundler Audit

## Interface Direction

Nexspro is built with a Ruby-first interface approach.

The view layer is designed to stay explicit, composable, and maintainable. Phlex allows the project to keep the interface structured as Ruby code while preserving the speed and simplicity of server-rendered Rails.

The visual system is intentionally restrained: focused typography, controlled spacing, consistent rhythm, clear navigation, and a strong sense of identity.

## Engineering Principles

Nexspro is developed around a few core principles:

- Prefer Rails conventions before custom abstractions.
- Keep the application understandable as it grows.
- Use Phlex for main application views.
- Keep rendering Ruby-first and server-driven.
- Avoid unnecessary JavaScript.
- Build components with clear ownership.
- Maintain a consistent visual and technical system.
- Protect the product identity.
- Prioritize long-term maintainability.

## Repository Status

This repository contains the active foundation of Nexspro.

It includes the application structure, routing, layout system, Phlex views, components, visual identity, assets, and public-facing implementation currently under development.

The project is expected to grow significantly over time.
